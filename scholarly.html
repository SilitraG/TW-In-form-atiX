Informatix

1. Introducere

1.1 Motivație

Motivatia acestui proiect este una intuitiva deoarece fiind student in cadrul facultatii de Informatica, o tematica asemeni celei prezentate in aceasta documentatie este de mare interes. Site-ul urmareste in primul rand indeplinirea cerintelor atent stabilite care se pot vizualiza la sectiunea proiecte din cadrul site.ului oferit drept suport de studiu pentru materia Tehnologii Web. De asemenea un astfel de site vine in ajutorul oricarei persoane ce doreste sa lucreze suplimentar probleme de informatica.

1.2 Scopul produsului

Scopul acestui produs este oferirea unui mediu interactiv, stabil, sigur si propice pentru dezvoltarea personala a aptitudinilor de programare ale oricarui elev de liceu sau de ce nu, student universitar. Site-ul urmareste indeaproape nevoile utilizatorilor punand la dispozitie sectiuni de Q&A si de evaluare a problemelor rezolvate.

2. Descriere generală

2.1 Perspectiva Produsului

Similar cu alte site-uri ce permit vizualizarea si rezolvarea problemelor, Informatix pune la dispozitie o gama larga de probleme si exercitii precum si statistici despre solutiile trimise si punctajele obtinute

2.2 Funcțiile Produsului

Aplicatia Web ofera urmatoarele facilitati:

  • Formular de inregistrare respectiv logare, ambele oferind doua roluri distincte, cel de elev si cel de profesor
  • Propunerea unor probele noi de catre userii de tip Profesor
  • Aprobarea acestor probleme si listarea lor pe site de catre userii de tip Admin
  • Crearea de clase noi de catre profesori in cadrul carora se pot adauga elevi dupa diversi identificatori (email, username, etc.)
  • Propunerea anumitor teme catre o astfel de clasa
  • Incarcarea solutiilor la problemele propuse pentru userii de tip elev
  • Adaugarea comentariilor la probleme din partea elevilor ce doresc sa lase indicatii despre rezolvare altor elevi.
  • Acces la formular de feedback trimis automat catre un email prestabilit in codul sursa al site.ului
  • Acces atat de catre elevi cat si de catre profesori la o gama larga de statistici

2.3 Tipuri de utilizatori

  1. Utilizator neinregistrat
    • Are dreptul de a naviga pe pagina principala a site.ului
    • Poate accesa pagini precum ‘About’, ‘Help’, ‘Login’, ‘Register’
  2. Utilizator inregistrat
    • Ofera un feedback despre site oricand doreste
    • Vizualizeaza sectiunea de Q&A cu intrebari frecvent adresate

    Student/Elev

    • Poate accesa probleme de pe site
    • Primeste invitatie in clase, ca mai apoi sa dea join oricand doreste
    • Poate vizualiza statistici despre propriile probleme rezolvate

    Profesor

    • Creaza clase noi si invita elevii in aceste clase
    • Propune teme catre elevii deja existenti intr-o clasa
    • Propune probleme noi care pot fi listate catre rezolvare pe site
  3. Admin
    • Poate aproba problemele noi propuse de catre profesori
    • Poate raspunde oricarei intrebari primita in sectiunea de feedback
    • Poate modela si prelucra datele de pe site in orice mod doreste

    2.4 Mediu de utilizare

    Informatix este o aplicatie web ce poate fi accesata utilizand orice browser web.

    2.5 Constrangeri de Design si Implementare

    3. Functionalitati de Sistem

    Site.ul pune la dispozitie urmatoarele pagini:

    1. Home - pe aceasta pagina sunt directionati userii ce acceseaza link.ul site.ului Informatix. In partea de sus gasim meniul principal al aplicatiei iar la doar un scroll distanta, cele 3 categorii principale de probleme pe care site.ul le pune la dispozitia elevilor, fiecare dintre ele cu cateva statistici utile.
    2. Login - pagina ce permite accesarea contului, indiferent daca este vorba despre elevi sau profesori
    3. Register - aici se pot crea conturi noi, cu optiunea de a alege tipul de user
    4. About - permite utilizatorilor sa descopere o mica parte despre cei ce au stat la crearea acestui site
    5. Help - Aici putem gasi intrebari frecvent adresate cu raspunsuri. De asemenea, prin intermediul unui formular de feedback, se poate adresa orice mesaj de sustinere, recomandare de functionalitate noua sau intrebare direct administatorilor acestui site

    3.1 Individualizarea paginilor CSS Style

    In aceasta sectiune trebuie mentionat faptul ca fiecare pagina mai sus descrisa are propria pagina de CSS Style, unde au fost implementate toate detaliile ce tin de design.ul acestora

    3.2 Statistici

    Modulul de backend al acestui site prelucreaza datele existente in bazele de date si pune la dispozitie o gama larga de statistici, atat generale despre numarul de probleme rezolvate, despre numarul userilor inregistrati in sistem, etc. cat si despre fiecare user in parte (Numarul de solutii trimise in cadrul unei probleme, punctajul mediu obtinut, categoria preferata, etc.)

    3.3 Organizarea problemelor in subcategorii

    Pentru o navigare mai usoara pe site, problemele au tag.uri ce le permit mai apoi sa fie stocate pe grupuri si sectiuni. De exemplu, in categoria de probleme usoare regasim subcategoriile de Divizibilitate, Cifrele unui numar, etc.

    3.4 Search bar/Filtrare

    Ca si bonus fata de cerinta existenta, punem la dispozitie o bara de cautare si un buton de filtrare in functie de tag.urile fiecarei probleme. Aceste functionalitati usureaza navigarea utilizatorilor prin multitudinea de probleme puse la dispozitie.
